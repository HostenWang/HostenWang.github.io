进程的最大文件描述符数量偏少|Max file descriptors for process likely too low

  1. HostenWang

    Guest Author

1 概述|Overview

记录 Max file descriptors for process likely too low 问题解决

2 环境|Environment

3 报错

3.1 Apache NiFi

max file descriptors [2048] for nifi process likely too low, increase to at least [65536]

3.2 Elasticsearch

max file descriptors [4096] for elasticsearch process likely too low, increase to at least [65536]

4 解决

# 修改切换到root用户
> su

# 修改配置limits.conf 添加下面两行
$> vim /etc/security/limits.conf
\*        hard    nofile           65536
\*        soft    nofile           65536

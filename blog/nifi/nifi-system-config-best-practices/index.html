Apache NiFi 系统配置最佳实践|Apache NiFi System Configuration Best Practices

  1. HostenWang

    Guest Author

概述|Overview

如果您在Linux上运行，请考虑以下最佳实践。

典型的Linux缺省值并不一定能够很好地适应像NiFi这样的IO密集型应用程序的需要。

文件句柄数量调优|Maximum File Handles

NiFi在任何时候都可能打开大量的文件句柄

root> vim /etc/security/limits.conf
* hard nofile 50000
* soft nofile 50000

进程数量调优|Maximum Forked Processes

NiFi配置为生成大量线程。要增加允许的数量

root> vim /etc/security/limits.con
* hard nproc 10000
* soft nproc 10000

root> vim /etc/security/limits.d/90-nproc.conf
* soft nproc 10000

TCP socket调优|TCP socket Tunning

如果您的流程将在短时间内设置和拆除大量套接字

# Increase the number of TCP socket ports available
root> sudo sysctl -w net.ipv4.ip_local_port_range="10000 65000"

# Set how long sockets stay in a TIMED_WAIT state when closed
root> sudo sysctl -w net.ipv4.netfilter.ip_conntrack_tcp_timeout_time_wait="1"

禁用 Swapping|Disable Swapping

Swapping 对某些应用很好，但不适用一直在运行的NIFI，

root> /etc/sysctl.conf
vm.swappiness = 0

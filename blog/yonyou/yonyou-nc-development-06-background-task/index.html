概述|Overview

本文记录如何在 用友(yonyou) NC 使用后台任务功能

后台任务是合理利用系统资源、提升系统运行效率的重要途径，大部分耗时的、周期性处理的业务处理过程都可以部署成后台任务来实现

环境|Environment

名称 版本 备注
Windows 10 home x64
jdk8 131以上
U8Cloud 3.1
eclipse_u8c

开发

业务插件类要求实现了nc.bs.pub.taskcenter.IBackgroundWorkPlugin接口

该接口方法 executeTask中写入定时任务要执行的逻辑，出于人性化考虑，在return的时候返回字符串应该能够让客户明白该定时任务的执行情况，以前项目中有些同事的定时任务往往没返回任务成功或失败的提示，以至于客户一直认为系统在健康的运行着。

public class MaintenanceAlart implements IBackgroundWorkPlugin {

    public String Syu8cAPIPerForTest(BgWorkingContext bgwc) throws BusinessException {

    // 写需要执行的模块
        return '成功处理数据XXX条'
	}
}

注册

部署

#部署 涉及如下设置

触发设置：控制后台任务的发生时间，有即时和定时两种。根据需求选择立即执行或定时（测试时建议选择立即执行）

消息接收配置：定义后台任务以何种方式发送给用户，有消息中心（默认）、邮件、短信三种。（可以不管）

日志和监控

日志

后台任务 #日志 显示了任务的每次执行结果，其中结果列显示了该次执行的结果（成功/失败），如果失败，说明列显示了错误信息。

该处是对于通过任务中心部署的任务执行结果进行日志，并不是对调度引擎所有执行的任务进行跟踪。主要记录任务实体的开始执行时间、结束执行时间、执行结果、说明、条目定义对象。

监控

后台任务中心提供对运行期的任务的 #监控 和管理

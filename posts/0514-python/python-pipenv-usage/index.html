Python pipenv 使用总结

Mar 08, 2023
阅读时长: 2 分钟

概述|Abstract

pipenv 是 Kenneth Reitz大神的作品，提供Python的各个版本间的管理，各种包管理

Pipenv is a Python virtualenv management tool that supports a multitude of systems and nicely bridges the gaps between pip, pyenv and virtualenv. Linux, macOS, and Windows are all first-class citizens in pipenv.

背景|Background

Pipenv 是 Kenneth Reitz 在2017年1月发布的Python依赖管理工具，现在由PyPA维护。你可以把它看做是pip和virtualenv的组合体，而它基于的Pipfile则用来替代旧的依赖记录方式（requirements.txt）。

安装

1

> pip install pipenv

换源

vim Pipfile

1
2
3
4
5

[[source]]

url = "https://mirrors.aliyun.com/pypi/simple"
verify_ssl = true
name = "pypi"

单次

1

> pipenv install --pypi-mirror https://mirrors.aliyun.com/pypi/simple

使用

创建虚拟环境

1
2
3
4

# OLD
> virtualenv venv
# NEW
> pipenv install

激活虚拟环境

1
2
3
4
5

# OLD
> venv\Scripts\activate
# NEW
> pipenv shell
> pipenv run

安装依赖到虚拟环境

1
2
3
4
5
6

# OLD
> venv\Scripts\activate
(venv)$ pip install flask requests

# NEW
> pipenv install flask flask-wtf

记录依赖

1
2
3
4
5

# OLD
(venv)$ pip freeze > requirements.txt

# NEW
> pipenv install flask flask-wtf

在部署环境安装依赖

1
2
3
4
5
6
7

# OLD
$ virtualenv venv  # 创建虚拟环境
$ . venv/bin/activate  # 激活虚拟环境
(venv)$ pip install -r requirements.txt  # 安装requirement.txt中记录的依赖

# NEW
> pipenv install

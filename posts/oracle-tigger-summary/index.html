Oracle Trigger Summary

By HostenWang

October 10, 2020

前言

当前项目里使用触发器做准实时同步,顺便小节一下.

简介

官方原文

A trigger is a named program unit that is stored in the database and fired (executed) in response to a specified event. The specified event is associated with either a table, a view, a schema, or the database, and it is one of the following:

  • A database manipulation (DML) statement (DELETE, INSERT, or UPDATE)
  • A database definition (DDL) statement (CREATE, ALTER, or DROP)
  • A database operation (SERVERERROR, LOGON, LOGOFF, STARTUP, or SHUTDOWN)

The trigger is said to be defined on the table, view, schema, or database.

触发器是保存在数据库里的有名称的代码单元,并在一些指定的事件后自动执行.能够触发触发器的事件有以下几种：

  • DML操作（INSERT、UPDATE、DELETE）
  • DDL操作（CREATE、ALTER、DROP）
  • 系统事件（数据库的关闭与启动等）
  • 用户事件（用户的登陆等）

使用场景

  1. 允许/限制对表的修改
  2. 自动生成派生列，比如自增字段
  3. 强制数据一致性
  4. 提供审计和日志记录
  5. 防止无效的事务处理
  6. 启用复杂的业务逻辑

触发器语法

-- 触发器名:Print_salary_changes
CREATE OR REPLACE TRIGGER Print_salary_changes
  -- BEFORE 表示在数据库动作之前触发器执行
  BEFORE DELETE OR INSERT OR UPDATE ON emp
  -- 对表的每一行触发器执行一次。如果没有这一选项，则只对整个表执行一次
  FOR EACH ROW
-- 触发条件
WHEN (NEW.EMPNO > 0)
-- 声明
DECLARE
    sal_diff number;
-- 触发动作,SQL语句
BEGIN
   -- sal_diff = 新值 - 旧值
    sal_diff  := :NEW.SAL  - :OLD.SAL;
    dbms_output.put('Old salary: ' || :OLD.sal);
    dbms_output.put('  New salary: ' || :NEW.sal);
    dbms_output.put_line('  Difference ' || sal_diff);
END;

注意事项

  1. 触发器不接受参数
  2. 在触发器的执行部分只能用DML语句（SELECT、INSERT、UPDATE、DELETE），不能使用DDL语句（CREATE、ALTER、DROP）
  3. 触发器中不能使用commit语句，触发器的操作与触发事件（INSERT,UPDATE,DELETE）一起进行COMMIT和ROLLBACK;
  4. 一个表上最多可有12个触发器，但同一时间、同一事件、同一类型的触发器只能有一个。并各触发器之间不能有矛盾
  5. 一个表上的触发器越多，对于表的DML操作性能影响越大
  6. 触发器最大为32KB。若确实需要，可以先建立过程，然后在触发器中用CALL语句进行调用
  7. 在触发器主体中不能申明任何Long和blob变量。新值new和旧值old也不能是表中的任何long和blob列

触发器的简单管理

操作 SQL语句
禁用触发器 alter trigger Print_salary_changes disable
启用触发器 alter trigger Print_salary_changes enable
删除触发器 drop trigger Print_salary_changes

java junit test 单元测试

Junit 总结|Junit Usage

命名规范,常用注解和断言

Posted by HostenWang on Monday, July 10, 2017

1 概述|Overview

JUnit 是一种单元测试框架

2 命名规范

测试类一般用被测试类+Tests来命名。

should 常用于测试方法名，如ordersShouldBeCreated，menuShouldGetActive，以增强可读性。

总之，测试名应该尽量表明测试进行的内容，而不必去看代码。

3 常用注解

常用 #注解 如下

@Test   //此方法为测试方法. 
public void method()

//标识可能抛出的异常
@Test (expected = Exception.class) 

//超过指定时间，测试失败。
@Test(timeout=100)  
    
//该方法在每次测试前运行，用来准备测试环境，如读取输入数据，初始化类等。 
@Before             
public void method()

//方法在测试后运行，用来清理，如删除临时文件，恢复初始设置，释放内存等。     
@After              
public void method()

//方法在所有测试前运行一次，用来执行费时任务，如连接数据库。必须是Static方法。 
@BeforeClass            
public static void method()    

//方法在所有测试完成后执行一次，用来清理，如断开数据库连接，必须是Static方法。 
@AfterClass             
public static void method()    

//忽略指定测试方法，用于测试类尚未准备好等情况，使用时最好标明忽略的原因。  
@Ignore or @Ignore("禁用原因")

4 常用断言

常用 #断言 如下

// 让方法失败。可用于确认某部分代码未被运行，或在测试类完成前人为返回失败结果，message参数可选。
fail(message) 

assertTrue([message,] boolean condition)    确认布尔为真。

assertFalse([message,] boolean condition)    确认布尔为假

// 确认返回=期望，数组等比较的是地址而非内容。
assertEquals([message,] expected, actual)    

assertEquals([message,] expected, actual, tolerance)    测试浮点数相等。

assertNull([message,] object)    对象为NULL

assertNotNull([message,] object)    对象非NULL

assertSame([message,] expected, actual)    变量指向同一个目标.

assertNotSame([message,] expected, actual)    变量指向不同目标

5 例程

@org.junit.Test
public void multiplicationOfZeroIntegersShouldReturnZero() {

   // MyClass is tested
   MyClass tester = new MyClass();

   // Tests
   assertEquals("1 x 0 must be 0", 0, tester.multiply(1, 0));
   assertEquals("0 x 1 must be 0", 0, tester.multiply(0, 1));
   assertEquals("0 x 0 must be 0", 0, tester.multiply(0, 0));
 }

PostGIS 创建索引|PostGIS Building Indexes

Posted by HostenWang on Wednesday, October 6, 2021

1 概述|Overview

PostGIS 是目前业界主流的地理数据库之一，提供如下空间信息服务功能：空间对象、空间索引、空间操作函数和空间操作符等。

本文介绍 #PostGIS 索引概念

2 PostGIS 中的空间索引

索引让空间数据库的大量数据查询成为可能，没有索引，任何特性的查询都需要进行全表扫描。索引通过将数据丢到一个遍历树中以便快速找到一个指定的记录。

PostgreSQL 默认支持三种索引类型：B-Tree索引，R-Tree索引和GiST索引

2.1 B-Tree 索引

GIS 中很少使用

2.2 R-Tree 索引

R-Tree 这种索引把数据分隔成矩形，子矩形（sub-rectangles）和子子矩形（sub-sub rectangles）等等。R-Tree 索引适用于部分空间数类型，但是PostgreSQL的R-Tree索引实现不如GiST索引稳健

R-Trees break up data into rectangles, and sub-rectangles, and sub-sub rectangles, etc. R-Trees are used by some spatial databases to index GIS data, but the PostgreSQL R-Tree implementation is not as robust as the GiST implementation.

2.3 GiST 索引

GiST (Generalized Search Trees缩写)索引把数据分成这样几个部分：某一边的对象，重叠的对象，内部的对象，该索引适用 范围很广，包括GIS数据。

GiST (Generalized Search Trees) indexes break up data into “things to one side”, “things which overlap”, “things which are inside” and can be used on a wide range of data-types, including GIS data. PostGIS uses an R-Tree index implemented on top of GiST to index GIS data.

3 使用索引

一般来说，索引能够隐式地加速获取数据：一旦创建了索引，查询优化器会决定什么时候使用索引信息来加速查询。

以下的语句给sde模式中的cities表添加了一个空间索引shape_index_cities，在pgAdmin中也可以通过图形界面完成相同的功能。

> CREATE INDEX shape_index_cities ON sde.cities USING gist (shape);

另外要注意的是，空间索引只有在进行基于边界范围的查询时才起作用，比如“&&”操作

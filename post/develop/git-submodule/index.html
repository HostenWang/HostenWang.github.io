Git submodule 使用总结|Git submodule Usage

Posted by HostenWang on Friday, May 26, 2017

1 概述|Overview

面对比较复杂的项目，我们需要将代码根据功能拆解成不同的子模块。主项目依赖于子模块，又不用关心子模块的内部开发流程细节, 此时就需要使用 git 的 submodule 功能.

git submodule 是一个很好的多项目使用共同仓库的工具，它允许仓库项目做为repository,子项目做为一个单独的git项目存在父项目中，子项目有自己的独立的commit，push，pull.

2 增加子模块|Add Submodule

# clone parent repo
> git clone https://github.com/demo/repo-1.git prj-main
> cd prj-main

# add submodule
> git submodule add -b main https://github.com/demo/repo-2.git prj-sub

# prj-sub is empty
> ls prj-sub
.
..
.git

# check git config files
> ls .git/modules/
prj-sub

> cat .gitmodule
[submodule "prj-sub"]
    path = prj-sub
    url = https://github.com/demo/repo-2.git

> cat .git/config
[submodule "prj-sub"]
    url = https://github.com/demo/repo-2.git
	
# update
> git submodule update --init --recursive

# now, prj-sub folder is not empty
> ls prj-sub
.
..
.git
READEM.md

3 更新|Update

如果该仓库作者进行了更新，我们需要手动进行更新。即进入子模块后，执行

> cd prj-sub
> git pull

4 删除|Delete

> git submodule deinit prj-sub
> git rm --cached prj-sub
> git rm prj-sub
> rm  -rf .git/modules/prj-sub

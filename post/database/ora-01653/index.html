ORA 01653 表空间无法拓展的解决办法|ORA 01653 unable to extend table

Posted by HostenWang on Friday, March 22, 2019

1 相关报错

ORA-00604: erroroccurred at recursive SQL level 1
ORA-01000: 超出打开游标的最大数 
ORA-01653: unable to extend table SYS.AUD$ by 8192 in tablespace
ORA-01653: unable toextend table SYS.AUD$ by 8192 in tablespace SYSTEM
ORA-02002: errorwhile writing to audit trail

2 排查

2.1 11g 查看表空间使用率,不推荐

-- 它的大小计算单位是block，需要关联下数据库的block大小
-- 表空间总容量如果是可自动扩展的datafile，那么结果就是所有datafile最大的max值的和
SQL> select * from dba_tablespace_usage_metrics;
TABLESPACE_NAME|USED_SPACE|TABLESPACE_SIZE|USED_PERCENT  |
SYSTEM         |    127240|        4194302|  3.033|
TEMP           |       128|        4194302|  0.003|
USERS          |       520|        4194302|  0.012|

SQL> show parameter db_block_size

2.2 手写 SQL查看表空间使用率,推荐,参考 1

2.2.1 永久表空间

select a.tablespace_name,
       a.member,
       round(a.max / 1024 / 1024 / 1024) "MAX_size GB",
       round((a.max - (a.bytes - b.bytes)) / 1024 / 1024 / 1024) "MAX_free GB",
       round((a.bytes - b.bytes) / 1024 / 1024 / 1024) "MAX_used GB",
       round((a.bytes - b.bytes) / a.max * 100) "MAX_used_percent %",
       round(a.bytes / 1024 / 1024) "Now_Sum MB",
       round((a.bytes - b.bytes) / 1024 / 1024) "Now_used MB",
       round(b.bytes / 1024 / 1024) "Now_free MB",
       round(((a.bytes - b.bytes) / a.bytes) * 100) "Now_used_percent %"
  from (select tablespace_name,sum(bytes) bytes,sum(decode(maxbytes, 0, bytes, maxbytes)) max,count(file_id) member from dba_data_files group by tablespace_name) a,
       (select tablespace_name,sum(bytes) bytes,max(bytes) largest from dba_free_space group by tablespace_name) b
 where a.tablespace_name = b.tablespace_name
 order by 5, 1;

2.2.2 临时表空间

select a.tablespace_name,
   count(b.file_id) member,
       sum(round(decode(b.maxbytes, 0, b.bytes, b.maxbytes) / 1024 / 1024 / 1024)) "MAX_size GB",
       sum(round(a.allocated_space / 1024 / 1024 / 1024)) "MAX_used GB",
       sum(round((decode(b.maxbytes, 0, b.bytes, b.maxbytes) - a.allocated_space) / 1024 / 1024 / 1024)) "MAX_free GB",
       sum(round(a.allocated_space / decode(b.maxbytes, 0, b.bytes, b.maxbytes) * 100)) "MAX_used_percent %",
       sum(a.TABLESPACE_SIZE / 1024 / 1024) "Now_sum MB",
       sum((a.TABLESPACE_SIZE - a.FREE_SPACE) / 1024 / 1024) "Now_used MB",
       sum(a.FREE_SPACE / 1024 / 1024) "Now_FREE MB",
       sum(a.ALLOCATED_SPACE / 1024 / 1024) "Now_ALLOCATED MB",
       sum(round((a.TABLESPACE_SIZE - a.FREE_SPACE) / a.TABLESPACE_SIZE * 100)) "Now_percent_used %"
  from dba_temp_free_space a, dba_temp_files b
 where a.tablespace_name = b.tablespace_name
 group by a.tablespace_name;

3 定位问题

发现 SYSTEM 表空间使用率达到 99.4%, 需要扩容

4 解决方案

4.1 联系 DBA 扩容

4.1.1 表空间对应的物理文件

SQL> SELECT T.TABLESPACE_NAME,D.FILE_NAME,D.AUTOEXTENSIBLE,D.BYTES,D.MAXBYTES,D.STATUS 
FROM DBA_TABLESPACES T,DBA_DATA_FILES D 
WHERE T.TABLESPACE_NAME =D.TABLESPACE_NAME 
ORDER BY TABLESPACE_NAME,FILE_NAME; 

TABLESPACE_NAME|FILE_NAME                          |AUTOEXTENSIBLE|STATUS   |
SYSTEM         |      F:\SUN\ORCL\SYSTEM01.DBF     |YES           |AVAILABLE|
SYSAUX         |      F:\SUN\ORCL\SYSAUX.DBF       |YES           |AVAILABLE|
USERS          |      F:\SUN\ORCL\USERS01.DBF      |YES           |AVAILABLE|

4.1.2 扩大表空间大小

-- 方案1: 给表空间增加数据文件，并允许自动增长：
SQL> ALTER TABLESPACE SYSTEM ADD DATAFILE 'F:\SUN\ORCL\SYSTEM02.DBF' SIZE 500M AUTOEXTEND ON NEXT 500M MAXSIZE 20000M;

-- 方案2: 手工改变已存在数据文件大小
SQL> alter database datafile 'F:\SUN\ORCL\SYSTEM01.DBF' autoextend on;
SQL> alter database datafile 'F:\SUN\ORCL\SYSTEM01.DBF' resize 1200M;

-- 方案3: 允许已存在的数据文件自动增长：
SQL> alter database datafile 'F:\SUN\ORCL\SYSTEM01.DBF' autoextend on next 500M maxsize unlimited;

4.1.3 掉 AUD$表并关闭审计功能


-- 清理审计表
SQL> truncate table SYS.AUD$;
SQL> show parameter audit;
-- --关闭审计功能
SQL> alter system set audit_trail=none scope=spfile;
SQL> shutdown immediate;
SQL> startup;

-- 验证
SQL> show parameter audit;

5 原因

系统表空间正常情况下只存放了数据字典之类的东西，所以占用的空间一般在500M以下。如果你的系统表空间占用比较多的空间，可能有以下几方面的原因：

1）没有为用户明确指定默认表空间，导致 SYSTEM系统表空间作为用户默认表空间

2）开启了审计,请检查此表的大小 AUD$

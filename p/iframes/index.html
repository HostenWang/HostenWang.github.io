HTML iframes 使用总结 | How and When to Use HTML IFrames

Jan 06, 2019
1 minute read

1 概述|Overview

我厂使用 #iframe 较多, 这里做个总结

我厂自研的 #快速开发框架 能自动生成数据库表的 list/add/edit 表单页面, 业务页面通过 iframe 加载 list/add/edit 页面即可实现功能的增查改删, 实现大量业务功能开发, 可以极大提高开发效率.

3 iframe 介绍

iframe是嵌入式框架, 是html标签, 还是一个内联元素, iframe 元素会创建包含另外一个文档的内联框架（即行内框架) . 说白了, iframe用来在页面嵌入其他页面.

通常我们使用iframe直接直接在页面嵌套iframe标签指定src就可以了。

4 优势劣势和使用场景

4.1 优势

  1. 隔离完美

    1. 独立展示嵌入的页面, 能够把嵌入的网页原样展现出来, 如果使用div嵌入会影响页面样式
    2. 异步加载.如果遇到加载缓慢的第三方内容如图标和广告，这些问题可以由iframe来解决.所以很多页面广告都基于 iframe
    3. 局部刷新.重载页面时不需要重载整个页面，只需要重载页面中的一个框架页

  2. 使用简单

    1. 代码复用，便于更改. 将公共页面放到 iframe 内加载

  3. ​

4.2 劣势

  1. 路由状态丢失

  2. 网页结构变复杂

    1. 通信困难
    2. 不利于SEO, 影响搜索引擎优化，不利于网站排名
    3. 不调优会影响用户体验；
    4. 增加网络请求.CSS和JS需要额外链入，调用外部页面, 需要额外调用css,增加页面额外的请求次数，增加服务器的http请求；
    5. 加载速度慢.HTML Window 的 onload 事件需要在所有 iframe 加载完毕后(包含里面的元素)才会触发，会影响网页加载速度。

  3. 设备兼容性差.多数小型的移动设备（PDA 手机）无法完全显示框架

4.3 使用场景

  1. 简单的后台界面
  2. 无刷新文件上传
  3. 跨域通信
  4. 独立的宿主环境

5 常用属性

  1. frameborder:是否显示边框，1(yes),0(no)
  2. height:建议在使用css设置。
  3. width:建议使用css设置。
  4. name:名称，window.frames[name]时专用的属性。
  5. scrolling:是否滚动。yes,no,auto。
  6. src：地址，可以使页面地址，也可以是图片的地址。

6.1 注意事项

要确保在Iframe加载完成后再进行操作，如果Iframe还未加载完成就开始调用里面的方法或变量会产生错误。判断Iframe是否加载完毕有两种方法：

  1. 在Iframe上用 onload 事件
  2. 用document.readyState=="complete"来判断

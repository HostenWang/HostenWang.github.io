高效使用 Axure的几条原则|Several Principes Of Using Axure Efficiently

高效使用 Axure的几条原则|Several Principes Of Using Axure Efficiently

Oct 05, 2022
1 minute read

概述|Overview

在互联网产品领域，Axure 已成为产品经理、产品设计师以及交互设计师的必备工具. Axure帮我们建立低保真模型，便于与用户的需求验证，也帮助我们构思交互细节，使前端和开发人员更容易理解我们的产品.

环境|Environment

名称 版本 备注
Windows 7 home x64

原则

原型不等于开发

  1. 不要陷入展现网页样式, 视觉效果等细节. 原型不等于开发
  2. 有些用笔记(Note)能说明的交互, 和研发沟通以后可以不画原型(如重复的增删改查操作), 聚焦确实需要原型进行说明的场景.

积累自己的组件库(Axure RP library)

如设计自己公司的表单组件, 如输入组件, 搜索组件等

尽可能使用母版 Master

使用母版的便利性

  • 能够快速的使用一组控件
  • 在设计后期快速调整组件

单个母版尽可能功能单一

不要把太多的组合对象变成母版。越是多的组合对象，越是有可能需要在母版的很多地方做出修改。把一些母版和另外一个母版合并起来一般会是更好的办法。

学会使用样式(Styles)

便于组件风格的统一

在创建母版之前要设置好样式

保持项目的组织性和命名的清晰性

刚开始会觉得多此一举, 后续原型复杂以后, 在设计交互动作时好的命名会让选择目标组件轻松很多

学会使用 this 对象

主要方便设置组件状态

总结|Summary

从短期来看，这些原则很少能显示出效果，但是从长远来看，能使你工作更高效。
